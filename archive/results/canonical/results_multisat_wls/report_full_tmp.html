6. 标准伪距观测方程

本次在导航层显式使用的标准伪距方程为：

pseudorange_m =
    geometric_range_m
    + c * (receiver_clock_bias_s - satellite_clock_bias_s)
    + tropo_delay_m
    + dispersive_delay_m
    + hardware_bias_m
    + noise_m

各项物理意义如下：

  • geometric_range_m：卫星与接收机之间的几何距离。
  • receiver_clock_bias_s：接收机钟差。
  • satellite_clock_bias_s：卫星钟差。
  • tropo_delay_m：对流层延迟，当前采用参数化模型。
  • dispersive_delay_m：类电离层 / 类等离子体传播项，这里优先挂接旧脚本真实 tau_g 的量级与背景。
  • hardware_bias_m：硬件偏差，当前采用参数化模型。
  • noise_m：测量噪声，当前用旧脚本单通道观测误差统计来定标。

必须强调：单通道 DLL 输出的 c * tau_est 是链路内部恢复的码时延量，它不是自动等同于导航层标准伪距。标准伪距必须进入几何、钟差、传播误差统一观测方程后，才能用于多星导航解算。

标准伪距形成的关键实现逻辑如下：

# notebooks/exp_multisat_wls_pvt_report.py
pseudorange_m = (
    geometric_range_m
    + receiver_cfg.receiver_clock_bias_m
    - C_LIGHT * sat.sat_clock_bias_s
    + tropo_delay_m
    + dispersive_delay_m
    + hardware_bias_m
    + noise_m
)

9. 从单通道背景到多星标准伪距

新脚本最关键的“桥接层”就是：如何从真实单通道背景构造多星标准伪距。

当前实现方式是：

  1. 多星几何由自洽的方位角/仰角场景生成。
  2. dispersive_delay_m 用旧链路真实 tau_g 的量级作为基底，再按仰角映射。
  3. sigma_m 不再凭空手写，而是从旧脚本单通道伪距误差 1 s 平滑 sigma 映射得到。
  4. 低仰角卫星会进一步放大 sigma 与非几何误差项。

这段逻辑对应的实现如下：

# notebooks/exp_multisat_wls_pvt_report.py
sigma_m = legacy_bg.effective_pseudorange_sigma_1s_m * config.sigma_scale_factor / (sin_el ** 0.70)
dispersive_delay_m = legacy_bg.tau_g_median_m * config.dispersive_scale_factor / (sin_el ** 1.25)

pseudorange_m = (
    geometric_range_m
    + receiver_cfg.receiver_clock_bias_m
    - C_LIGHT * sat.sat_clock_bias_s
    + tropo_delay_m
    + dispersive_delay_m
    + hardware_bias_m
    + noise_m
)

10. LS 与 WLS 单历元解算方法

本次单历元 PVT 的状态量定义为：

[x_m, y_m, z_m, receiver_clock_bias_m]

求解方法是迭代线性化。设计矩阵 H 的每一行来自对应卫星的 LOS 单位向量：

# notebooks/exp_multisat_wls_pvt_report.py
line_of_sight_m = obs.sat_pos_ecef_m - state_m[:3]
geometric_range_m = float(np.linalg.norm(line_of_sight_m))
los_unit = line_of_sight_m / geometric_range_m

h_rows.append([-los_unit[0], -los_unit[1], -los_unit[2], 1.0])
residual_vector_m.append(obs.pseudorange_m - predicted_pseudorange_m)

LS 与 WLS 的区别只体现在权阵：

if weighted:
    weight_matrix = np.diag(1.0 / np.maximum(sigma_m, 1e-6) ** 2)
    normal_matrix = h_matrix.T @ weight_matrix @ h_matrix
    rhs_vector = h_matrix.T @ weight_matrix @ residual_vector_m
else:
    normal_matrix = h_matrix.T @ h_matrix
    rhs_vector = h_matrix.T @ residual_vector_m

这意味着：

  • LS 对所有观测一视同仁。
  • WLS 会根据 sigma_m 自动降低低质量观测的影响。
  • 当低仰角卫星误差显著放大时，WLS 理论上应比 LS 更稳健。

11. 实验设计

11.1 实验 A

  • 目标：验证系统在较好几何和中等噪声下可正常工作。
  • 场景：8 星，仰角整体较高。
  • 预期：LS 和 WLS 都应收敛，WLS 小幅优于 LS。

11.2 实验 B

  • 目标：通过低仰角差观测放大 LS 的劣势，突出 WLS 的价值。
  • 场景：保留 8 星，但引入多个低仰角观测。
  • 预期：LS 明显变差，WLS 有显著改善。

11.3 实验 C

  • 目标：不只看单次 realization，而是看统计结果。
  • 场景：基于实验 B 进行 120 次 Monte Carlo。
  • 输出：LS/WLS 的 3D 误差分布、均值、标准差和 90% 分位数。

14. 当前仍然采用的简化与边界

这一节非常关键，因为它决定了汇报口径是否诚实。

14.1 当前仍然存在的简化

  1. 多星几何不是广播星历
    当前卫星位置由静态自洽的 az/el 场景构造，不是真实广播星历，也没有轨道动力学。

  2. 所有卫星共享同一真实单通道传播背景
    新脚本复用了旧文件真实单通道 WKB / 接收机背景，但并没有为每颗卫星单独生成不同的真实鞘套路径。

  3. 对流层、硬件偏差、卫星钟差是参数化项
    这些项目前是导航层新增模型，不来自旧文件真实链路。

  4. 多星观测方差来自单通道统计映射
    多星 sigma_m 是基于旧脚本单通道观测误差统计映射得到的，不是真实多星同步跟踪直接输出。

14.2 本次结论的适用边界

本报告的结论应被理解为：

真实单通道 Ka/WKB 背景耦合下的 hybrid 多星标准伪距 / WLS 导航层实验结果。

不应被理解为：

真实多星端到端 Ka 导航系统性能定标。

因此，本次的“好结果”与“坏结果”都应放在这个边界内解释。它们说明当前旧链路背景与多星导航层耦合后的表现，而不是最终系统性能上限。
